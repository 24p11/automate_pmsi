Command line tools¶

This package will also install a few command line utilities that can be ran directly from the terminal (rather than Python). For example:

$ grep_keras_progress notebook.ipynb

will parse the results from the progress bar generated by tf.keras and print them on the terminal in the CSV format (which can be copy-pasted into Google sheets).

train_model¶

Run NOTEBOOK from the command line and set parameters of notebook variables.

NOTEBOOK needs to have a cell with parameters tag as described in the documentation.

train_model [OPTIONS] NOTEBOOK

Options

-n, --n-epochs <n_epochs>¶

number of epochs (N_EPOCHS in notebook) [required]

-p, --parameter <cli_parameters>¶

set value of parameter PARAM to VALUE in the notebook, it can be used multiple times

--save-path <save_path>¶

top directory for sub-directories with model artifacts

--log-path <log_path>¶

path to save output notebook

--output-dir <output_dir>¶

path to save model artifacts (used to configure OUTPUT_DIR in the notebook). Defaults to SAVE_PATH/NOTEBOOK-PARAMETER-VALUE…

-r, --raw-parameter <raw_cli_parameters>¶

like -p but do not convert the type

-P, --silent-parameter <silent_cli_parameters>¶

like -p but do not use the parameter in the filenames

-l, --learning-rate <learning_rate>¶

learning rate (LEARNING_RATE variable in notebook)

-s, --suffix <suffix>¶

suffix to set in file names

-f, --force¶

force ovewrite the log file

-g, --gpu <gpu>¶

comma delimited list gpus to run the model on

-v, --verbose¶

make the output more verbose (progress bar, etc.)

Arguments

NOTEBOOK¶

Required argument

Note

NOTEBOOK must have one cell with the parameters tag. You can add the tag in the cell properties side bar as described here. There are several parameter names that are defined by default by the train_model tool and that you should use in your code:

  • VERBOSITY: verbosity level according to keras definition (1 for verbose, 2 for more silent)

  • N_EPOCHS: number of training epochs (set by -n parameter)

  • OUTPUT_DIR: directory where the model outputs are written

  • LEARNING_RATE: learning rate (set by -l parameter)

See also a sample notebook on GitHub. Use it as a template for your own projects.

download_hdfs_csv¶

Download spark-generated mulit-part CSV from HDFS and stich them together

usage: download_hdfs_csv [-h] [--has_header] input_file [output_file]
input_file¶

path on the HDFS file system, it must point to a directory with multiple CSV files

output_file¶

path to save the final CSV file to, prints to the terminal if not defined

-h, --help¶

show this help message and exit

--has_header¶

interpret first line of each CSV as header
